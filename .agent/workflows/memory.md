# メモリー管理ワークフロー (Mem0 版)

このワークフローは、プランからノートを整理し、Mem0 (OpenMemory) に保存するための手順を提供します。
Mem0 は自然言語から自動的にエンティティや関係性を抽出するため、構造化データではなく自然言語での入力が中心となります。

## 1. 会話の履歴、実施した作業からノートの整理

1. 現在の会話履歴を確認する
2. 重複した物はまとめて構いませんが、事実は要約をせず、別々に残してください。
3. 以下のカテゴリに分類する:
  - プロジェクト構造と設計方針
  - 技術的な課題と解決策
  - 実装の詳細と進捗状況
  - 今後の計画と優先事項
  - **アイデンティティ**: 氏名、年齢、所在地、職業などの基本情報。
  - **興味・関心**: 趣味、習慣、最近の話題など。
  - **好み**: コミュニケーションスタイル、専門用語の使用レベル、好みのフォーマットなど。
  - **目標**: 短期的・長期的なゴール、達成したいこと。
  - **関係性**: 頻繁に登場する人物、組織、プロジェクトとの関係。

## 2. Mem0 (OpenMemory) の更新

### 2.1 基本的な更新手順

**1. 既存のメモリを確認する**:

```
list_memories()
```

**2. 新しい記憶を追加する**:

Mem0 では、自然言語のテキストを渡すだけで、自動的にエンティティ、関係性、事実が抽出・保存されます。
「誰が」「何を」「どうした」という文脈を含めて記述することが重要です。

```
add_memories({
  text: "ユーザー den は WiredForest という組織のオーナーである。WiredForest は様々なインフラを利用している。"
})
```

**3. 複数の情報を追加する場合**:

```
add_memories({
  text: "whisperプロジェクトは音声から文字起こしを行うプロジェクトであり、Docker Compose を利用してデプロイされている。"
})
```

### 2.2 保存のコツ

#### 明示的な関係性の記述
Mem0 は文脈から関係性を推論します。
- 悪い例: "WiredForest。オーナー。"
- 良い例: "den は WiredForest のオーナーとして働いている。"

#### 継続的な学習
整理したノートの中から、以下の情報を自然言語の文章として保存します：

- **アイデンティティ**: "den は東京に住むソフトウェアエンジニアである。"
- **好み**: "den は簡潔な回答を好み、日本語でのコミュニケーションを希望している。"
- **目標**: "den は Mem0 を使って自身の AI エージェントに長期記憶を持たせることを目指している。"

## 3. 確認と検証

**1. 保存されたメモリを検索して確認する:**

```
search_memory({ query: "WiredForest" })

# または特定のトピックについて検索
search_memory({ query: "den の好み" })
```

**2. 必要に応じて追加情報を投入する**
検索結果が不十分な場合は、より詳細な文脈を含めて `add_memories` を実行してください。
